Transit Citizen

The world needs well crafted open-source transit software. Why pay contractors to build rebuild our transit IT infrastructure with our tax dollars for each city when we could create an open set of solutions that any city can afford to implement?

It's time to take matters into our own hands

Take Action

Transit is a basic Need

Tansit's #1 use is to help people get to work, and we all deserve to have system we can both rely on and afford.

We deserve better

Transit systems around the world are custom built for each city. This causes a great expense to taxpayers, and make it impossible for systems to build on one another.

Anyone can help

By open sourcing all the necessary software for a city's public transit system, anyone who works on this project will be comtributing to the health of communities worldwide.

What's Needed

Transit infrastructure is complex, but meeting a city's needs in basic ways will create systems that many communities can adopt.

Needed Features

Open Requirement

We need to push cities with current transit system to publish their software requirements.

Fleet Inventory

Cities need to be able to track vehicle data, including service logs, route history, up vs. downtime.

Rider Validation

Vehicles need to be able to validate a rider's farecard or monthly pass. This includes magnetic stripe cards, NFC enabled passes, and new technologies.

Vehicle Tracking

Staying on schedule requires knowing where vehicles are in the transit network. This information is useful for updating route capacity, logging actual vehicle miles, and informing allowing the public to track wait times.

Open Hardware

From farecard readers to GPS tracking, most of the in-vehicle technology needed to make the boarding and tracking experiences run smoothly are availble to hobbyists and present in modern smartphones.

Route Definitions

Transit systems need to be able to plan routes, schedule waypoint times, define service frequency & intervals, and publish information for riders to use in both digital and printed forms.

Outward Facing Data

Transit system need to publish route, tracking, and other data to websites, apis, and mobile apps for 3rd party and public consumption.

Customer Service

Riders need help with issues from purchasing fares, to tracking a vehicle, to reporting incendents.

Payment Processing

Riders need many options for purchasing fares, from cash on the vehicle, all the way to recharging the balance of an NFC eneable transit card.
